guessXMLRPCURLForSite:success:failure:

Given a site URL, tries to guess the URL for the XML-RPC endpoint

+ (void)guessXMLRPCURLForSite:(NSURL *)siteURL success:(void ( ^ ) ( NSURL *siteURL , NSURL *xmlrpcURL ))success failure:(void ( ^ ) ( NSURL *siteURL ))failure

Parameters

siteURL

The site’s main url, e.g.: http://en.blog.wordpress.com/

success

A block object to execute when the method finds a suitable XML-RPC endpoint on the site provided. This block has no return value and takes two arguments: the original site URL, and the found XML-RPC endpoint URL.

failure

A block object to execute when the method doesn’t find a suitable XML-RPC endpoint on the site. This block has no return value and takes one argument: the original site URL.

Discussion

Given a site URL, tries to guess the URL for the XML-RPC endpoint

publishPostWithGallery:description:title:success:failure:

Publishes a post asynchronously with an image gallery

- (void)publishPostWithGallery:(NSArray *)images description:(NSString *)content title:(NSString *)title success:(void ( ^ ) ( NSUInteger postId , NSURL *permalink ))success failure:(void ( ^ ) ( NSError *error ))failure

Parameters

images

An array containing images (as UIImage) to add to the post. The gallery will be embedded before the content using the [gallery] shortcode.

content

The post content/body. It can be text only or HTML, but be aware that some HTML might be stripped in WordPress. What’s allowed in WordPress.com?

title

The post title.

success

A block object to execute when the method successfully publishes the post. This block has no return value and takes two arguments: the resulting post ID, and the permalink (or shortlink if available).

failure

A block object to execute when the method can’t publish the post. This block has no return value and takes one argument: a NSError object with details on the error.

Discussion

Publishes a post asynchronously with an image gallery

All the parameters are optional, and can be set to nil

publishPostWithVideo:description:title:success:failure:

Publishes a post asynchronously with a video

- (void)publishPostWithVideo:(NSString *)videoPath description:(NSString *)content title:(NSString *)title success:(void ( ^ ) ( NSUInteger postId , NSURL *permalink ))success failure:(void ( ^ ) ( NSError *error ))failure

Parameters

videoPath

A string containing the path to the video file to add to the post. The video will be embedded before the content.

content

The post content/body. It can be text only or HTML, but be aware that some HTML might be stripped in WordPress. What’s allowed in WordPress.com?

title

The post title.

success

A block object to execute when the method successfully publishes the post. This block has no return value and takes two arguments: the resulting post ID, and the permalink (or shortlink if available).

failure

A block object to execute when the method can’t publish the post. This block has no return value and takes one argument: a NSError object with details on the error.

Discussion

Publishes a post asynchronously with a video

All the parameters are optional, and can be set to nil

For WordPress.com, if VideoPress is not available for the blog, there will be an error.

For self hosted blogs, if VideoPress is available it will be used, otherwise the video will be embedded using the HTML5 <video> tag.
